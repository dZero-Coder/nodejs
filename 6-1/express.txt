1. Express-generator 전역설치
    → npm i -g express-generator

2. 새 익스프레스 프로젝트 생성
    → express [프로젝트명] --view=pug
    (--view=pug : 템플릿 엔진을 Pug를 설치)

3. express 프로젝트에 npm, 모듈 설치
    → cd [프로젝트명] && npm i

-------------------------------------------------------
서버 실행 : npm start

< 폴더 구조 >
- app.js : 핵심적인 서버 역할

bin/www : 서버를 실행하는 스크립트
    - http 모듈에 express 모듈을 연결하고, 포트를 지정하는 부분
    - www 파일을 콘솔 명령어로 만들 수 있음

- public/ : 외부에서 접근 가능한 파일 (image, csss, js 파일)
- routes/ : 주소별 라우터들을 모아둔 곳 (서버 로직)
- views/ : 템플릿 파일을 모아둔 곳 (화면 부분)
- model/ : DB 관련 파일(생성해야함)


< 미들웨어 >
- express의 핵심으로 요청과 응답의 중간에 위치하여 미들웨어라고 부름
- 라우터와 에러 핸들러 또한 미들웨어의 일종
- 요청과 응답을 조작하여 기능을 추가하거나 나쁜 요청을 걸러냄
- app.use()의 인자로 들어있는 함수가 미들웨어
